# Github Action

## push to another repository

### ci.yml

```yaml
name: Push Selected Files to Another Repository

on:
  push:
    branches:
      - dev

jobs:
  push:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4
        with:
          ref: dev

      - name: Get Commit Message
        id: get_commit_message
        run: |
          COMMIT_MESSAGE=$(git log -1 --pretty=%B)
          # 写入环境变量，之后可以直接使用
          echo "commit_message=$COMMIT_MESSAGE" >> $GITHUB_ENV

      - name: Copy specific folder
        run: |
          if [ -d "../tmp" ]; then rm -rf ../tmp; fi
          mkdir -p ../tmp
          rsync -av --inplace --delete \
                    --exclude='.git/' \
                    --exclude='.github/workflows/ci.yml' \
                    --exclude='docs/.obsidian/' \
                    --exclude='docs/WorkNote/' \
                    --exclude='docs/其他知识/阅读与日记/日记/' \
                    ./ ../tmp/
          mv ../tmp ./

      # 勉强能实现我想要的提交记录，提交记录是针对smileatl/smileatlRepo的变化
      - name: Push to Another Repository
        uses: cpina/github-action-push-to-another-repository@main
        env:
          API_TOKEN_GITHUB: ${{ secrets.ACCESS_TOKEN }}
        with:
          source-directory: "tmp" 
          destination-github-username: "smileatl"
          destination-repository-name: "smileatlRepo"
          user-email: "songlei.lin@qq.com"
          target-branch: master
          # 也可以使用"${{ steps.<id>.outputs.<variable> }}"来获取
          commit-message: "${{ env.commit_message }}"
```

## pull request触发了push事件

**push事件触发**：

- 直接push到该分支
- 合并了一个pull request

**pull_request事件会在以下情况下触发**：

- **创建 Pull Request**：当一个新的 Pull Request 被创建时。
- **更新 Pull Request**：当现有的 Pull Request 被更新时，包括提交新的代码、修改 Pull Request 的标题或描述。
- **重新打开 Pull Request**：当一个已关闭的 Pull Request 被重新打开时。
- **同步 Pull Request**：当 Pull Request 的源分支有新的提交时（例如，通过推送新的提交到源分支）。
